## Dikkat Edilmesi Gerekenler

- **`.widget { position: relative }` Gridstack item'a uygulandığında BOZAR** — `.grid-stack-item { position: absolute }` Gridstack CSS kuralını specificity'de override ediyor, widget'lar block-flow olarak diagonal akar. V2'de bu kurali kaldırdım. Aynı tuzak V1 builder'da da olabilir, kontrol etmeli.
- **SpPreview ile Reports/Run davranış farkı:**
  - SpPreview: `INFORMATION_SCHEMA.PARAMETERS` üzerinden TÜM SP param'larını listeleyip tip-bazlı default doldurur (date→today, int→0, nvarchar→'')
  - Reports/Run: SADECE ParamSchema'daki field'ları SP'ye geçirir, diğer SP param'ları NULL kalır → SP kendi default'unu kullanır
  - SP'lerde `IF @param IS NOT NULL THEN filter ELSE all` pattern'i varsa, SpPreview default `''` ya da `0` filter'i tetikler ve veri keser. Bu davranış SpPreview için F-11 smart defaults kapsamında düzeltilmeli (default'ları NULL yap, override yoksa SP'nin default'una bırak).
- **Razor `@@click`** — Alpine `@click` için Razor'da `@@click` (double-@) escape şart. Tüm V2 view'larda doğru.
- **Razor view cache** — preview server runtime hot reload yapar ama yeni view bazen browser cache'inden eski hâli yükler. dotnet build + preview restart cache reset eder.
- **Preview Chrome viewport farkı** — preview_resize default 1060px gibi dar açılıyor; mode-seg/builder shell test için 1440x900 manual resize gerekiyor.
