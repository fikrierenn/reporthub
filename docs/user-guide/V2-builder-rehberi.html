1. Genel Akış

V2 builder; SQL stored procedure'ünden gelen 7 result set'lik veriyi sürükle-bırak widget'larla dashboard'a dönüştürmenizi sağlar. Tipik akış 5-10 dakika alır:

┌────────────┐ ┌────────────┐ ┌────────────┐ ┌────────────┐ │ 1. Yeni │ → │ 2. Genel + │ → │ 3. Tasarım │ → │ 4. Kaydet │ │ Rapor │ │ Teknik │ │ (V2) │ │ │ │ klasik form│ │ form'u dol-│ │ widget + │ │ DB'ye yaz │ │ ile başla │ │ dur │ │ bağla │ │ │ └────────────┘ └────────────┘ └────────────┘ └────────────┘
Önemli
Her widget bir SQL stored procedure'inin belirli bir kolonuna bağlanır. Sistem hangi kolon ne anlama geldiğini bilemez — anlamlandırmayı siz yapın: veri setlerine isim verin, kolon kartlarına tıklayıp widget'lara bağlayın.

2. V2 Builder'a Erişim

  1. Sol menü → Yönetim Paneli'ne girin (admin yetkisi gerekir).
  2. Raporlar sekmesinde mevcut bir rapor seçin veya + Yeni Rapor ile başlayın.
    Klasik formda Genel Bilgiler + Teknik Ayarlar + Param Schema'yı doldurun:
    • Rapor Adı, Veri Kaynağı, Açıklama, Kategoriler
    • Prosedür Adı (örn. sp_PdksPano)
    • İzinli Roller, ParamSchemaJson (Tarih, vs.)
  3. Kaydet sonrası adres çubuğunda URL'yi EditReport → EditReportV2 olarak değiştirin:
    http://<host>/Admin/EditReportV2/<reportId>
Geçici durum
V2 ile V1 sayfaları paralel çalışıyor. Kategori/Rol/Param Schema gibi metadata'yı klasik formda düzenliyorsunuz; tasarımı ise V2'de. Sonraki sürümde tek sayfada birleşecek.

4. Veri Setlerini Tanıma

Sayfa açıldığında stored procedure otomatik çalıştırılır (ParamSchema'daki default değerlerle, örn. Tarih: bugün). Dönen result set'ler:

  • Sol palette'te Veri Setleri bölümünde
  • Sağ drawer Veri sekmesinde detaylı (kolon listesi + örnek değer)

Bir SP genellikle 3-7 result set döndürür:

Veri Seti 1 50 satır · 18 kolon → Plan-Fiili kişi listesi (örn.) Veri Seti 2 7 satır · 10 kolon → Şube özeti Veri Seti 3 1 satır · 14 kolon → Toplam KPI'lar Veri Seti 4 10 satır · 6 kolon → Fazla mesai listesi Veri Seti 5 10 satır · 5 kolon → Geç kalanlar Veri Seti 6 13 satır · 4 kolon → Bölüm doluluk Veri Seti 7 15 satır · 5 kolon → Eksik okutmalar

5. Veri Setlerine İsim Verme

Default isim "Veri Seti 1, 2, 3..." → kullanıcılarınıza anlamlı gelmez. Anlamlı etiketler verin:

  1. Sol palette → Veri Setleri başlığının altındaki listeye gidin.
  2. "Veri Seti 1" yazısına tıklayın → düzenlenebilir input olur.
  3. Anlamlı bir ad yazın (örn. "Plan-Fiili Detay", "Genel KPI", "Şube Özet").
  4. Enter veya alanı terk edince kaydedilir.
  5. Aynı işlemi sağ drawer Veri sekmesinde de yapabilirsiniz (orada büyük başlık olarak gösterilir).
Sonuç
Verilen isimler DashboardConfigJson.resultContract alanına kaydedilir; sayfayı kapayıp açan herkes (ve sonraki kullanıcılar) bu adları görür.
📷
EKRAN: Veri Setleri başlık düzenleme

6. Widget Ekleme

Sol palette'in en üstünde 3 widget tipi:

TipNe içinDefault boyut
KPITek sayı + delta (örn. "Toplam Kadro: 254")3×2 (1/4 genişlik, 2 satır)
GrafikTrend / Bar / Pie8×4 (2/3 genişlik, 4 satır)
TabloResult set satırlarını liste4×4 (1/3 genişlik, 4 satır)

Eklemenin 3 yolu:

  • Tek tık: Palette kartına tıkla → canvas'ta otomatik bir konuma yerleştirilir
  • + butonu: Kartın sağ alt köşesindeki + ikonu
  • Sürükle-bırak: Karta basılı tut → canvas'ta istediğin yere bırak

Yerleştirme + Boyutlandırma

  • Widget başlığını tutup sürükleyerek konum değiştir
  • Köşe veya kenardan tutup boyutlandır (Gridstack 12-col grid'e snap'lenir)
  • Layout otomatik kaydedilir (Kaydet'e tıklayınca DB'ye yazılır)

7. Veri Bağlama (1-tıkla)

Bu V2'nin en önemli özelliği. Widget'a hangi kolonun bağlı olacağını tek tıkla seçersiniz:

  1. Canvas'ta widget'ı tıklayın (seçili olduğunu kırmızı kenardan anlarsınız).
  2. Sağ drawer otomatik Veri sekmesinde açıktır.
  3. Drawer'da kolon kartı listesi görünür. Her kart şunları gösterir:
    • Sube · METİN · FSM ← örnek değer
    • KadroToplam · SAYI · 254
    • Tarih · TARİH · 2026-04-29
  4. İstediğiniz kolona tıklayın → seçili widget'a bağlanır:
    • comp.column ← kolon adı
    • comp.result ← result set adı (rs5)
    • comp.agg ← otomatik (sayı→sum, metin→count, tarih→first)
    • comp.title ← widget yeni eklenmişse kolon adına eşitlenir
  5. Bağlandığını üç yerden anlarsınız:
    • Drawer üstünde Bağlı: rs5 · Bolum [×] banner'ı
    • Widget header'ında result-pill: Veri Seti 6 · Bolum
    • Widget body'sinde bound-chip: 🔗 Bolum
Hızlı yol
Hiç widget seçili değilken bir kolon kartına tıklarsanız sistem otomatik olarak yeni KPI ekler ve o kolona bağlar. Boş canvas'tan başlarken zaman kazandırır.

Bağı Kaldırmak

Drawer Veri tab üstündeki "Bağlı: ... [×]" banner'ında × butonuna tıklayın.

Kolon Arama

SP çok kolon döndürüyorsa drawer'ın üstündeki "Kolon ara..." kutusuna yazın → tüm RS gruplarında filtre uygulanır.

9. Aggregation ve Formül (KPI için)

Drawer Veri tab'da en altta "İleri ayar ▸" bölümünü açın:

Aggregation (Hesap Yöntemi)

Tek bir kolon üzerinde 6 hesap tipi:

HesapAnlamıNe zaman
Satır SayısıResult set'teki satır sayısı"Toplam personel"
Toplam (sum)Kolon değerlerinin toplamı"Aylık ciro"
Ortalama (avg)Aritmetik ortalama"Ortalama sepet"
MinimumEn küçük değer"En düşük satış"
MaksimumEn büyük değer"En yüksek satış"
İlk Değer (first)İlk satırın değeri1-satırlık result set'ler

Formül (2-kolon)

Mevcut iki kolonu birleştirerek hesap yapın:

  1. Kolon A: ilk operand (RS gruplarına göre dropdown)
  2. İşlem: A + B, A − B, A × B, A ÷ B, % (A/B) (yüzde)
  3. Kolon B: ikinci operand

Örnekler:

  • Plan açığı: PlanToplam − PlanGeldi = bugün gelmemiş plan
  • Katılım yüzdesi: % (KadroGelen / KadroToplam) = devam yüzdesi
  • Ortalama sepet: Ciro ÷ FisAdeti
  • Trend farkı: BuAy − GecenAy
Sınırlamalar (V2)
Şu an sadece 2-kolon arası aritmetik destekleniyor. IF, ROUND, COALESCE, üçten fazla operand gibi karmaşık formüller için tam AST parser ileri sürümde gelecek.

10. Param-bar (Tarih Test)

Canvas'ın üstünde Param-bar görürsünüz:

┌────────────────────────────────────────────────────────────────┐ │ PARAMETRELER 📅 Tarih: 29.04.2026 🔄 Sıfırla ▶ Çalıştır │ └────────────────────────────────────────────────────────────────┘
  1. Tarih chip'ine tıklayın → date input açılır.
  2. İstediğiniz tarihi seçin (geçmiş tarihler için sayfayı yenilemenize gerek yok).
  3. Enter tuşu veya Çalıştır butonuna basın → SP yeni param ile çalıştırılır, widget değerleri yenilenir.
  4. Default'a dönmek için Sıfırla linkine tıklayın.
Neden işe yarar
Bugünün verisi henüz tamamlanmamış olabilir (gece, gün başlangıcı). Geçmiş tamamlanmış bir gün üzerinde dashboard'unuzun tüm değerlerini test edebilirsiniz. Param-bar değişiklikleri sadece test içindir, kaydedilmez.

11. Önizle / Düzenle

Topbar'da [Düzenle | Önizle] mode toggle:

Düzenle ModeÖnizle Mode
  • Drag-resize aktif
  • Palette + Drawer açık
  • Widget'lar "—" placeholder gösterir
  • Bound-chip görünür
  • Drag-resize pasif
  • Palette + Drawer dimmed
  • Widget'lar gerçek SP verisiyle dolu
  • Chart SVG ve tablo satırları render

Önizle modu, kullanıcının dashboard'u nasıl göreceğini tahmin etmenizi sağlar.

12. Kaydet

Topbar'ın sağ köşesindeki kırmızı Kaydet butonu, tüm değişiklikleri DB'ye yazar:

  • Widget listesi + boyutlar + bağlama
  • Veri Setleri için verdiğiniz isimler (Result Contract)
  • Renk + ikon + alt yazı + formül
  • Genel rapor metadata (klasik formdan gelmesi gerekiyorsa)

Kayıt sonrası sayfa yenilenir. Adres çubuğunda ?reportId=... parametresi varsa edit modu kaybolmaz.

Saklama yeri
Tüm tasarım ReportCatalog.DashboardConfigJson kolununa JSON olarak yazılır. SQL'de saklanır, tüm kullanıcılar aynı tasarımı görür.

JSON Kopyala

Topbar'da JSON butonuna tıklayınca DashboardConfigJson içeriği panoya kopyalanır. Backup için veya başka rapora aktarmak için kullanışlıdır.

13. Sık Sorulan Sorular

Soru: Widget'ı yanlış konuma yerleştirdim, eski hale nasıl alırım?

Cevap: Şu an undo (geri al) yok. Widget'ı tekrar sürükleyerek doğru yere taşıyın veya boyutlandırın. (V3 plan: Ctrl+Z destek)

Soru: KPI değeri "—" gösteriyor, neden?

Olası sebepler:

  1. Düzenle modundasınız (sadece preview moda gerçek değer görünür)
  2. Kolon henüz bağlanmamış (drawer'da bound banner yok)
  3. SP o tarih için 0 satır döndürüyor (param-bar'dan farklı tarih deneyin)
  4. Bağlı kolon NULL değer döndürüyor (SP'ye bakın)

Soru: Sayfa açılınca veri çok yavaş geliyor.

SP çağrısı arka planda gerçekleşir. Result set'lerde 50+ satır cap'ı vardır (UI sınırı). SP'nin kendisi yavaşsa optimizasyon SP tarafında yapılmalı (index, sub-query, vs).

Soru: Bir widget'ı silmek istiyorum.

Widget üzerinde mouse'unuzu gezdirin → sağ üst köşede 🗑 (çöp) ikonu görünür → tıklayın.

Soru: Aynı widget'ı başka yerde kullanmak istiyorum.

Widget üzerinde 📋 (kopyala) ikonu var → klona oluşturur, taşıyın.

Soru: Stored procedure'ün dönüş yapısı değişti, ne olur?

Yeni kolon → drawer'da otomatik görünür, bağlanmış widget'lar etkilenmez.
Eski kolon silindi → o kolona bağlı widget "—" gösterir; başka kolon seçin veya silin.
Result set sırası değişti → otomatik yeni başlık ("Veri Seti N") gelir; verdiğiniz isim (resultContract) rsN indeks'ine bağlı, isim eski indeksde kalır. Manual güncelleme gerekebilir.

Soru: V1 (klasik) sayfasıyla aynı raporu açtım, V2 değişikliklerimi görür mü?

Evet — ikisi de aynı DashboardConfigJson'a yazıyor. V1 sayfasını refreshlerseniz V2'de yaptığınız tasarımı görürsünüz.

Soru: Param-bar'da Tarih dışında başka param ekleyebilir miyim?

Evet — klasik form'daki ParamSchemaJson alanına yeni field ekleyin ("name", "type" = date/text/number/decimal/checkbox). V2 builder otomatik chip render eder.

Soru: SP'mde 7 result set var ama bazıları boş geliyor.

Bu SP'nin davranışı, V2 builder bunu değiştirmez. Eğer beklediğiniz veriyi aldığınız bir parametreyle test ederseniz (param-bar'da farklı tarih deneyin), boş RS'lerin gerçek SP davranışı mı yoksa default param sorunu mu olduğunu anlarsınız.

Yardım?
Bu rehberle çözülmeyen durumlar için sistem yöneticinize başvurun. SQL tarafındaki SP veya yetki konuları, builder'ın değil; backend tarafının sorunudur.
